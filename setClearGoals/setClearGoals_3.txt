Автоматизированные тесты должны быть:

    - Поддерживаемыми - для снижения количества усилий по тех. обслуживанию
    - Актуальными - для обеспечения четкой бизнес-ценности автоматизации
    - Многоразовыми - для обеспечения модульности тестов
    - Управляемыми - для обеспечения эффективного проектирования, выполнения и прослеживаемости
    - Доступными - для обеспечения совместной работы по параллельному проектированию и разработке
    - Надежными - для обеспечения обработки и восстановления объекта/события/ошибок
    - Портативными - быть независимыми от SUT и быть полностью масштабируемыми
    - Надежными - для обеспечения отказоустойчивости
    - Диагностируемыми - для выявления дефектов для ускоренного исследования дефектов
    - Измеримый - предоставьте панель тестирования вместе с настраиваемой отчетностью

Потенциальные проблемы:
    Обязательно напомните руководству, что автоматизировать каждый тест невозможно или целесообразно.
    Отсутствие измеримых целей. Особенно таких как "найти много дефектов" при автоматизации регрессионных тестов.
